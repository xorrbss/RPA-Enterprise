개발 메모

웹사이트를 조작하면서 클릭, 입력, 선택, 검증을 녹화하고 셀렉터 후보를 저장하는 진입점을 추가합니다.

완료 기준

녹화한 액션이 실행 가능한 시나리오 초안으로 저장되고, 셀렉터 테스트 결과를 사용자가 확인할 수 있습니다.

추천 개발 순서

데스크톱 제외 후에도 즉시 개발 검토할 만한 1차 흐름입니다.

1
지원하지 않는 액션의 혼선 제거

api_call, file, shell은 브라우저 전용 범위 밖이면 작성/저장 단계에서 차단하거나 명확히 비활성 처리합니다.

P0
2

P0
3

P0
4

승인뿐 아니라 데이터 보정, timeout, 담당자 재배정까지 닫힌 업무 흐름으로 만듭니다.

P1
5

P1

반영 재분류 제외
반영P0

레코더와 PbD 승격

브라우저 RPA의 제작 진입점을 낮추고, 성공 실행을 재사용 가능한 봇으로 바꿉니다.

반영P1

HITL 업무 데이터 입력

사람 확인을 단순 계속 신호가 아니라 검수, 보정, 승인 데이터를 받는 업무 화면으로 확장합니다.

반영P1

Challenge 복구

MFA, CAPTCHA, 로그인 만료를 감지한 뒤 사람 확인과 세션 갱신, 재시도를 연결합니다.

재분류P0

비지원 액션 혼선 제거

비브라우저 액션을 당장 구현하기보다, 사용자가 실행 가능한 것처럼 착각하지 않게 막습니다.

재분류P2

웹 요소 저장소

Object Repository를 Windows 객체가 아니라 사이트별 셀렉터 라이브러리로 정의합니다.

제외Out

VDI, Citrix, Windows 자동화

현재 제품 포지션이 브라우저 전용이면 경쟁 비교표에서 제외하고 별도 라인업으로 분리합니다.

개발 판단 메모: 이 화면의 핵심은 “경쟁사 기능을 모두 따라 하자”가 아닙니다. 브라우저 전용 제품이라고 명확히 선을 긋되, 그 범위 안에서 도입자가 바로 막히는 제작, 예외, 보안, 감사 경험을 먼저 닫는 방향입니다.
