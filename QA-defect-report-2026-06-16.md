## 총평

코드베이스의 **자동화 회귀 게이트(CI)는 전부 green**이다(빌드·타입체크·단위/통합/e2e 통과). 그러나 **실제 사용자 흐름은 핵심에서 무너진다.** 사용자가 "자동화 만들기 → 승격 → 실행"을 끝까지 따라가도, 임의 사이트(쿠팡)에서는 **실행이 조용히 죽고 그 사실이 UI에 전혀 표시되지 않는다.** 이 프로젝트가 문서로 공언한 핵심 원칙 **"조용한 false/unknown 금지(no silent false)"** 가 정작 **사용자 대면 경로에서 깨져 있다**는 점이 가장 큰 문제다.

> **범위 주의(공정성):** 일부 결함은 **dev 하니스**(`app/dev/*`, "테스트용·프로덕션 아님")에서 비롯된다(LLM 미배선, 데모 site_profile만 시드). 그러나 UI/UX·에러 표면화·백엔드 422 등 대부분은 **제품 코드(`web/src`, `app/src/api`)** 에 있으며 하니스와 무관하다. 아래 각 항목에 출처를 구분 표기했다.

## 상세

### D1 — [P0] 실패한 실행이 "점유"로 위장되고, 실패 이유가 전혀 노출되지 않음
- **재현:** 임의 URL(쿠팡)로 자동화 생성 → 실행 → 실행 기록에서 해당 run 확인 → "상세" 클릭.
- **현상:** run이 **"점유"(claimed) 상태로 영구 정지**. "실행 상세" 패널에 표시되는 건 `상태=점유 / 워커 / 시도 횟수=0 / 기준 시각`뿐 — **단계 트레이스도, 실패 이유도, 에러 코드도 없음.** 화면 부제는 "자동화가 어떤 순서로 판단하고 실행했는지 조회"라고 약속한다.
- **근거:** 런루프가 실행 전 단계에서 던진 `SiteResolutionError`를 잡아 **서버 콘솔에만** 로그하고 run은 claimed에 남긴다 — `app/dev/run-loop.ts:138-141`. 운영 콘솔의 run 상세(`web/src/views/RunTrace.tsx`)는 상태/워커/시도/as_of만 노출.
- **사용자 영향:** 자동화가 실패했는데 사용자는 그 사실조차 모른다. "조용한 false 금지"(CLAUDE.md 핵심 규칙) 정면 위반.
- **권고:** run에 종결-실패 전이(`failed_system` + reason_code)를 적용하고, run 상세에 실패 사유·마지막 단계·에러 코드를 표면화.

### D2 — [P0] 사용자가 만든 임의 사이트 자동화는 실행 자체가 막혀 있음 (사일런트)
- **재현:** "자동화 만들기"로 외부 URL 자동화 생성 → 승격 → 실행 → 아무 일도 안 일어남.
- **근거 체인:**
  1. `POST /v1/runs` 로 run 생성(queued) — 성공.
  2. 런루프가 claim 후 **실행 전 site_profile 사전점검**: `app/dev/run-loop.ts:115-119`.
  3. `resolveSiteProfileId` 가 `site_profiles` 에서 URL origin 매칭 → 0건 → `throw SITE_PROFILE_UNRESOLVED` (`app/src/runtime/site-resolution.ts:152-153`). **브라우저 navigate는 호출조차 안 됨.**
- **3중 차단:** ① 대상 사이트용 `site_profile`(페이지상태 셀렉터) 미등록 → 사전점검 실패. ② **콘솔에 site_profile을 등록하는 화면이 없음**(보안/사이트는 "승인"만) → 사용자가 되게 만들 방법 부재. ③ 시나리오에 `extract`(LLM) 단계가 있으면 dev 런루프가 `UtilityExecutor`(navigate/observe만)라 **LLM 미배선**으로 또 실패.
- **사용자 영향:** 데모 픽스처 외의 **실제 사이트 자동화는 콘솔만으로는 불가능**. 모든 실패가 사일런트(D1).
- **권고:** (제품) 사용자가 대상 사이트를 등록/설정하는 흐름 제공 + 실행 전 사전점검 실패를 UI로 표면화. (하니스) 실 LLM 실행기 배선 또는 "이 빌드에서 실행 불가" 명시.

### D3 — [P1] "테스트 실행" 화면에 실행 버튼이 없음
- **재현:** 좌측 "테스트 실행" → 자동화 선택.
- **현상:** 제목/부제가 *"만든 자동화를 안전하게 **시험 실행**하고 실제 실행 전 결과를 확인"* 인데, 화면엔 **정적 실행계획 미리보기 + 안내문**만 있고 **실행/dry-run 버튼이 없다.** 본문도 "실제 dry-run은 worker/Chrome 연결이 필요해 별도"라며 그 "별도" 경로로 가는 링크조차 없음.
- **사용자 영향:** 메뉴 이름이 약속한 기능이 부재(데드 기능). 사용자는 시험 실행을 할 수 없다.
- **권고:** 실제 dry-run 트리거를 연결하거나, 화면 명칭을 "실행 계획 미리보기"로 정정.

### D4 — [P1] AI 모델 설정 — gateway policy 로드가 422로 실패하고 에러 미표시
- **재현:** 좌측 "AI 모델 설정" 진입(또는 아무 화면에서나 콘솔 에러 확인).
- **현상:** `GET /api/v1/gateway/policy` 가 **422 Unprocessable Entity**를 반환(재시도로 4~5회). 시드에 정책 2개(gpt-4o-mini, claude-haiku)가 있는데도 화면엔 *"여러 모델 정책이 있습니다. 모델을 선택하면 상세가 표시됩니다"* 문구만 뜨고 **선택 목록도, 에러 안내도 없음.**
- **사용자 영향:** AI 모델 설정 기능 전체가 무력화. 실패가 사일런트(또 "no silent false" 위반).
- **권고:** 백엔드 422 원인 수정(목록 GET이 422를 내는 것 자체가 비정상) + 프런트에 에러/빈상태 표면화.

### D5 — [P2] prod 승격 버튼이 완료 후에도 무한 재클릭 가능
- **현상:** "prod 승격" → 확인 → "완료". 그러나 버튼이 비활성화되지 않아 **계속 다시 누를 수 있음.** 목록에 **promotion_status(prod/draft) 컬럼이 없어** 이미 승격됐는지조차 안 보임.
- **근거:** `web/src/views/Scenarios.tsx:52-58` — 승격 버튼에 "이미 prod" 가드/비활성 없음, 목록 컬럼은 이름/버전/버전ID/작업뿐.
- **권고:** 승격된 버전은 버튼 비활성 + 상태 뱃지(prod) 표시.

### D6 — [P2] 실행 다이얼로그가 이미 있는 URL을 다시 입력 요구 + 키/값 누수
- **현상:** "실행" 팝업에서 **거대한 대상 URL이 입력 필드의 "라벨(이름)"로 박혀** 있고, 그 아래 **빈 입력칸이 같은 URL을 다시 요구**한다. (사용자가 생성 시 URL을 `url_ref`의 값이 아닌 키 자리에 넣게 방치된 결과)
- **근거:** `web/src/components/RunScenarioButton.tsx:76-88` — `extractUrlRefKeys(ir)` 결과 `{k}` 를 라벨로 렌더하고 그 값을 다시 입력받음. 생성 시 입력값 prefill 없음.
- **권고:** url_ref는 사람이 읽는 이름으로 제한(검증), 생성 시 URL을 값으로 저장·prefill, "키 vs 값" 개념을 사용자에게 노출하지 않음.

### D7 — [P2] 긴 URL이 word-break 없이 페이지를 가로로 2배 오버플로
- **현상:** 쿼리스트링 긴 URL을 다루면 텍스트가 안 끊겨 **앱 전체 폭이 약 2배**가 되고 가로 스크롤이 생김(실행 팝업·테스트실행 미리보기 등에서 재현).
- **근거:** 긴 URL이 라벨/`이동(...)` 텍스트로 그대로 렌더되며 `overflow-wrap/word-break` 미적용.
- **권고:** URL 표시에 `overflow-wrap:anywhere`/말줄임 + 표는 `table-layout:fixed`.

### D8 — [P2] 사이트·담당자가 이름 대신 잘린 UUID로 표시되어 구분 불가
- **현상:** 보안/개인정보의 사이트 표가 이름("red-site"/"데모 사이트" 등) 대신 **8자 잘린 UUID**(70000000, 7a000000…)를 보여줘 **여러 행이 동일("7a000000")하게 보임.** 사람 확인의 "담당자"도 동일하게 UUID.
- **권고:** 식별 컬럼에 사람이 읽는 이름(또는 이름+짧은 id) 표시.

### D9 — [P2] "중복 방지" 메뉴가 빈 "준비 중" 스텁 + 내부 테이블명 노출
- **현상:** 사이드바에 노출된 "중복 방지" 화면이 *"준비 중 / control_plane_idempotency_keys read 표면이 노출되면 연결됩니다"* 한 줄. 미구현 메뉴가 운영 콘솔에 그대로 노출되고, **내부 DB 테이블명을 사용자에게 노출.**
- **권고:** 미구현 메뉴 숨김 또는 사용자 친화 빈상태 문구(내부 식별자 비노출).

### D10 — [P3] 비기술 운영자 콘솔에 개발자/계약용 화면이 그대로 노출 (altitude 미스매치)
- **현상:** "시나리오 검사"는 **시나리오 UUID + IR(JSON) 직접 입력** 폼이고(검사 실행 버튼 기본 비활성), "Product-open 점검"은 **endpoint·error code·RBAC 매트릭스** 계약 문서를 그대로 노출. 콘솔의 표방 대상은 비기술 한국어 운영자.
- **권고:** 개발자/계약용 화면은 별도 관리자/디버그 영역으로 분리하거나 운영자 친화적으로 추상화.

### D11 — "에이전트로 자동화하니 UI 불친절해도 괜찮다"는 변명은 성립하지 않음
- **Swagger/API 문서:** 스펙 파일(`codegen/openapi.yaml`, OpenAPI 3.1)은 존재하나, **실행 중인 서버가 Swagger UI/스펙을 서빙하지 않음**(`app/src` 전체에 swagger/scalar/redoc/`/docs` 서빙 0건). 에이전트가 라이브로 발견할 문서가 없다.
- **에이전트 전용 표면:** `/v1/agent/*` 같은 에이전트 지향 API **없음**, **MCP 서버도 없음**. 범용 운영 제어평면 REST일 뿐.
- **결정타:** 에이전트가 API로 run을 만들어도 **D2의 동일한 엔진 데드엔드**에 빠진다. 즉 사람이 UI를 쓰든 에이전트가 API를 쓰든 **실사이트 자동화는 동일하게 실패**한다.

### D12 — [P2] 사이트 등록·헤드리스 제어가 제품 표면에 전혀 없음 (운영 스위치 부재)
- **사이트 등록:** 자동화의 전제인 `site_profile`(URL 패턴 + `page_state_selectors` CSS 셀렉터)을 만드는 경로가 **UI에도 API에도 없음.** Sites 엔드포인트는 `GET /v1/sites`·`GET /v1/sites/{id}`·`POST /v1/sites/{id}/approve` 3개뿐(생성 없음, `app/src/api/sites.ts`는 approve만). **유일한 방법은 DB 직접 INSERT**(`dev/serve.ts` 시드 방식). → 운영자가 새 사이트 자동화를 스스로 등록 불가.
- **헤드리스 토글:** `createStagehandSession`은 `headless?: boolean`을 받지만(`app/src/executor/cdp-session.ts:136,160`), **모든 실행 경로가 `headless: true` 하드코딩**(`app/dev/run-loop.ts:71` 등). `HEADLESS` env·config·UI 토글 **전무**. 실사이트 봇 차단 시 headful로 전환할 운영 수단이 없다(코드 수정만이 유일).
- **사용자 영향:** "실사이트 RPA"를 표방하지만, 실사이트를 등록하거나 봇 차단에 대응하는 **운영 스위치가 0**. 둘 다 제품 기능이 아니라 개발자 코드/DBA 작업.

## 공정성 — 정상 동작하는 것
- 시드된 **데모(내장 픽스처) 자동화는 실제로 queued→completed까지 완주**(실 Chrome).
- **대시보드 / 작업 목록 / 사람 확인 / DLQ 재처리(W10) / 사이트 승인** 등 읽기·명령 흐름은 실 API↔PostgreSQL로 동작.

## 결론
회귀 게이트는 통과하지만 **end-to-end 사용자 가치는 미완성**이다. 특히 **(D1) 사일런트 실패와 (D2) 실사이트 실행 불가**가 제품의 본질(임의 사이트 RPA)을 막고 있으며, 둘 다 콘솔에 표면화되지 않는다. 우선순위: **D1·D2(P0) → D3·D4(P1) → 나머지.**
